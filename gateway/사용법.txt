장치에 블루투스 장비가 있고 켜져있는지 확인해주세요

1.
처음 시작 시, devices.txt 파일을 읽어 장치가 몇 개 있는지 파악합니다. 실행 중에 devices.txt를 수정해면 반영되지 않으니 수정 사항이 있으면 실행 중단 후 수정 바랍니다.

2.
다음과 같은 기능이 있습니다.
quit - 종료
scan - 장치를 탐색합니다. 장치를 사용할 준비가 되었다면 ONLINE, 그렇지 않다면 offline으로 표시됩니다.
list - scan을 통해 탐색한 장치의 목록을 출력합니다. scan을 따로 하진 않습니다.
get - 장치로부터 정해진 시간동안 IMU 데이터를 전송받습니다. 시간 경과 시, sensor.csv 에 센싱 기록을 저장합니다.(센서이름 알파벳 순서로 저장됩니다)
predict - 자세 추론을 수행합니다. 학습이 이미 완료된 모델 파일이 필요합니다.

**
만약 측정 중에 센서와 연결이 끊어져도, 측정은 끝나지 않습니다. 하지만 데이터는 연결이 끊어진 이후로 기록되지 않으므로, 바로 중지해 낭비되는 시간을 아껴야 합니다. 센서와 연결이 해제되면 연결이 해제되었다는 메시지가 출력되니, **터미널 화면을 잘 지켜보다** 이런 상황이 발생할 경우 KeyboardInterrupt(Ctrl+C)를 날려 실행을 중지하세요. 측정이 갑자기 중단된 경우, tempfile.csv 파일에 이제까지 수집한 데이터를 저장하게 됩니다.

async def time_indicate() 함수를 수정해, 측정이 진행되는 동안 터미널에 원하는 문장을 print를 할 수 있습니다.

센서는 마지막으로 연결이 해제된 후 10분 뒤에 꺼집니다. "get" 작업을 하기 전 센서를 모두 착용한 상태에서 reset버튼을 통해 센서를 모두 깨운 뒤 "scan"을 한번 더 돌린 후 "get"을 해주기 바랍니다.
